career-ops é gratuito e de código aberto (MIT). A única coisa que precisa ter é um **motor de IA** — um assistente de codificação baseado em IA que lê suas instruções e executa o trabalho. Esse motor também pode ser gratuito.

Você traz seu próprio motor e sua própria chave. Nada roda nos nossos servidores — tudo acontece na sua máquina. Esta página mostra os caminhos gratuitos, do mais fácil ao mais interativo. Escolha um e você estará pronto para o [Início Rápido](/docs).
<Callout title="O que estou realmente configurando?">
Pense em "career-ops" como a receita e o motor de IA como o cozinheiro. A receita é gratuita para sempre. Esta página se trata de obter um cozinheiro gratuitamente — seja um modelo hospedado com uma camada gratuita ou um modelo executando-se totalmente em seu próprio computador.
</Callout>

## Caminho 1 — OpenCode + um provedor gratuito (recomendado)

[OpenCode](https://opencode.ai) é um assistente de codificação AI open-source e gratuito. A visão de Santiago: *"open code é excelente."* Ele se conecta a vários provedores que oferecem uma **camada gratuita**, para que você possa executar as operações necessárias sem gastar nada.

## Caminho 3 — 100% local com Ollama (totalmente offline)

Quer zero nuvem, zero custo e total privacidade? Execute o modelo na sua própria máquina com [Ollama](https://ollama.com). Nada sai do seu computador.

O trade-off é o hardware. O career-ops pede ao modelo para avaliar vagas em diversas dimensões e personalizar seu currículo — modelos menores têm dificuldades com isso.
<Callout type="warn" title="Escolha um modelo que seja grande o suficiente">
Ignore os modelos pequenos de 7–8B — eles não atendem ao formato de avaliação e produzem CVs genéricos. Use um modelo de **32B ou maior** (por exemplo, Qwen 2.5 Coder 32B), que requer uma GPU com **16–24GB de VRAM** (uma RTX 3090/4090 ou um Mac Apple Silicon com 32GB+ de memória unificada). Sem essa máquina? A opção 1 é a rota gratuita.
</Callout>

```bash title="Terminal"
# Install Ollama from ollama.com, then pull a capable model:
ollama pull qwen2.5-coder:32b
```

Em seguida, direcione seu CLI de IA (OpenCode funciona bem aqui) para o endpoint local do Ollama, ou use o avaliador local integrado descrito no [guia de orçamento](https://github.com/santifer/career-ops/blob/main/docs/RUNNING_ON_A_BUDGET.md).## Caminho 4 — Atalho de um único comando: `npm run or`

O `career-ops` fornece um executável integrado que encaminha para os modelos gratuitos do **OpenRouter com fallback automático** — sem necessidade de configuração da linha de comando. Depois de clonar o `career-ops` e configurar uma chave do OpenRouter, um único comando executa o pipeline:

```bash title="Terminal"
export OPENROUTER_API_KEY="your_free_key_here"
npm run or            # runs the full pipeline on free models
```

Existem variantes focadas também — `npm run or:scan`, `or:eval`, `or:apply` — para executar uma única etapa.
